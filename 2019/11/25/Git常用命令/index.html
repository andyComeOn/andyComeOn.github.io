已不止一次看到上面截图提示！心想先不管它，先干活。我就在 dev 分支上修改了一个文件，于是 git status、git add .、git commit -m "修改 .gitignore 文件"、git pull、git push。push 的时候出现提示如下：

push 没成功，则必须要解决图中提示的问题，于是找到：git pull 报错 There is no tracking information for the current branch…。按照文中提示，我执行 git branch --set-upstream-to=origin/dev dev。随后又通过 git branch -vv 确认是否关联。总的截图如下：

上图则说明关联成功，git pull 拉下来很多代码！随后我又重复 git add .、git commit -m "修改 .gitignore 文件"、git pull、git push。把我刚才修改的文件 push 到 dev 了！此场景提示，说明在切换新的分支之后，若想使用简单明了 git pull 、git push，这必须事先把该分支，与远程的分支进行关联。否则，只能把指令写完整 git pull origin dev、git push origin dev。

2、查看、切换分支

  • 查看分支是 git branch 后面可以加参数 -a （指的是所有的分支）

  • 切换分支 git checkout 分支名

  • 创建分支 git checkout -b dev，此指令创建一个 dev 分支，若是 dev 分支已经存在，命令行会有提示：dev has already exists！

  • 创建于切换其实是一个指令，仅仅是添加的参数不同！

记录

  • 使用 webstorm 自带 git 工具 commit 时，提示如下：

意思：让我配置 crlf 自动转换，也就是说，本地的文格式是 crlf，git 远程仓库格式 lf，你本地的文件提交远程仓库需要转换！于是在该目录下执行了：git config --local core.autocrlf true，再次 commit 时候不在提示。

  • 在我的 files 项目，我在 commit 时候提示如下：

意思：将会被 autocrlf，不让我 commit，于是我执行 git config --local core.autocrlf false，vscode，webstorm 怎么提示的设置的不一样？原因：新建了 baby.html(其格式是 crlf)，然 git commit 默认将文件转为 lf，可是本地项目没有对该仓库进行配置，所以提示。若是大的工程项目，其下有一个.editorconfig 文件（在此文件中我们设置 end_of_line = lf），则在 git commit 时不会有警告！由这两种情况我们可得出，针对一个仓库若配置 end_of_line = lf，IDE 在 commit 时，也不会出现警告；若是简单小项目则可对该仓库设置：git config --local core.autocrlf true也不会出现警告；与其针对每个仓库执行上面的配置，我们可全局配置 git config --global core.autocrlf true。按理说，我们大概率将 core.autocrlf 设置为 true，可在部署 hexo 博客时候，只有设置为 core.autocrlf false 才让发布，具体 err 详情在“Hexo 杂集”博客中有提现。
