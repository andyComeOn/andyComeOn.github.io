vue-cli杂集

@vue/cli(3.0、4.0) 指令集

  • yarn add less-loader@5.0.0 less@4.1.1

前言

vue-cli 这是之前 2.0 的叫法，最后的版本是 2.9.6，其在 npm 的仓库地址在我的 Chrome 书签 vue 分类里。其 3.0 名字修改为@vue/cli，更加智能了，可配置 vue.config.js，在开发创值回顾 2.0 后，我决心看一下 3.0 之后的版本，看的时候已经更新到 v4.5.12 了。

过程

  • 代码颜色问题（v4.3-prettier vue 文件，template 没有换行，颜色不对）：

  • eslint 多余空行提示问题（v4.3-prettier 多余空行效果）：

  • eslint 多余空行提示问题（v4.3-prettier 安装安装依赖会提示你本地文件没有提交问题）：

  • 在 v4.3-prettier 中，我想使用 less，于是搜到vue-cli3/4 引入 less 全局变量
    文档中让执行 vue add style-resources-loader，后来发现在 vue-cli 的官方文档中有涉及。安装完了，vue.config.js 被创建。于是在 views/B.vue 中对 div 写 less 但是报错，细看 vue-cli 文档，在 style-resources-loader 文档前，讲述到“预处理器”，我于是执行 vue add less-loader less,提示没有找到该包名，截图如下：
    ![v4.3-prettier vue-add less-loader–less-err](https://file.798run.top/img/blog/20210402/vue-add less-loader–less-err.png)
    最开始猜想是不是淘宝源没有该依赖包呢？于是配置 yarn 的包仓库，再次执行依旧如上图提示。后来发现不应该使用 vue add，要使用 yarn add，于是yarn add less-loader less，启动项目发现报错如下：
    TypeError: this.getOptions is not a function,发现是 less-loader 的版本太高了，于是yarn remove less-loader，执行yarn add less-loader@5.0.0，启动项目可以了。
