Nodejs之Yarn

前言

yarn 安装有三种方式（官网说），最便捷的还是使用 npm i yarn@1.22.4 -g，默认安装目录：C:\Users\admin\AppData\Roaming\npm\yarn（Win 下执行 where yarn 可知）。yarn 诞生就是为了解决 npm 早期依赖包比较混乱的问题，伴随着 node 的升级，npm 版本也随之升级，针对依赖包已经做的很好了，详情可点击我。

指令

1、根据上面疑问版块的内容，可知，yarn 指令有：

  • yarn add pkgName
  • yarn add pkgName@1.22.4
  • yarn remove pkgName
  • yarn global add pkgName
  • yarn global remove pkgName
  • yarn config ls
  • yarn global ls

2、通用指令：

  • yarn config [options]
  • yarn global [options]

其实很多的指令其实就是通用指令加上子命令而来！假如我们记不清楚指令了，我们可键入 yarn global 会提示你缺少子命令，同时把相关常用的子命令给你罗列出来！截图如下：

看到上面的截图中的子命令有 bin ，执行 yarn global bin => C:\Users\admin\AppData\Local\Yarn\bin，结论：把此目录添加到环境变量，则使用 yarn 安装的全局依赖，在任意 cmd 下，就能使用了！

3、yarn upgrade 仅更新 yarn.lock、package.json 不更新，参考：升级包到它们基于规范范围的最新版本、yarn upgrade 更新依赖包时 yarn.lock 更新但 package.json 不同步更新版本信息

问题

1、在使用 yarn 安装依赖时候，一直提示：There appears to be trouble with your network connection. Retrying… 后来 Google 一下，看到一个帖子说的很好，总结原因：Ⅰ、网速不好，yarn 认为你有超时可能，于是提示你。Ⅱ、说执行 yarn config delete https-proxy && yarn config delete proxy。我认为是第一种原因最大，我已经在自己目录 ~ 下，创建 .yarnrc 文件，并设置为：network-timeout 600000，问题解决！
