React生命周期

用法

1、使用 react 时要注意其生命周期，最开始初始化 defaultProps、state。我们可以在组件的生命周期钩子函数中写相关逻辑来满足我们的业务需求。

2、componentWillMount：最初该钩子是为了异步的获取数据，现在官方已不赞成使用（在 ssr 场景下，服务端与客户端都会渲染一遍，浪费资源！）

3、componentDidMount：该组件是渲染完毕之后会触发！此时页面中已经有相关的 DOM，我们可以进行 DOM 的相关操作（思考：我们使用了 MVVM 之后还会操作 DOM 吗？）

4、我们在 componentDidMount 中可以己写一些逻辑。如：有一个 banner，我们需要给它动态设置 height，而 height 则是根据 width 而来，此时 dom 也已经渲染完了，我们可以获取该元素的 width，进而满足设置 height 的目的，最后在通过 setState 把 height 的值更新；若是组件的父组件的 prop 改变了，此时，就会走到 componentWillReceiveProps(newProps)，此钩子接受一个参数!

我们使用这个生命周期一般用于优化，有的时候，我们在界面上只需要更新一个很小的组件，而一个父组件的更新会造成整个子组件都进行渲染，形成一个崭新的虚拟 DOM，但是这样的话，会造成资源的浪费，我们可以根据实际的开发情况在 shouldComponentUpdate()生命周期加入条件，来进行性能的优化。

7、接着再次 render()，在接着是 componentDidUpdate，最后当离开页面时候，触发 componentWillUnMount，我们可以在 componentWillUnMount 中写清除定时器之类的操作！
