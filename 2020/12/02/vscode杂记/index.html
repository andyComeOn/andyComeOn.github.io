vscode杂记

前言：vscode 使用很广泛，记一些笔记

快捷键

1、md 文件，键入 Ctrl + Shift + v 可预览

▲ 放大缩小字体 Ctrl + (+/-)
▲ 显示/隐藏底部控制台：Ctrl + Shift + Y || Ctrl + ` 二则区别：前者是在 调试控制台 Tab 后者是在 终端 Tab
▲ 格式代码 => Shift + Alt + F
▲ 命令模式 => Ctrl + Shift + P （这里面可重载窗口、可选择 node_modules 插件查询下面的依赖包）
▲ 打开最近文件列表 => Ctrl + P || Ctrl + E
▲ 打开最近打开项目 => Ctrl + R ，按住 Ctrl 在鼠标点击项目，直接新窗口打开该项目

▲vscode 中添加很多项目时，鼠标选中某目录，直接 Shift + Alt + F 可直接检索出该目录下的检索词
▲ 资源管理器 => Ctrl + Shift + E 资源管理器(文件编辑区)
▲ 项目搜索 => Ctrl + Shift + F 项目搜索
▲ 在某文件搜索 => Ctrl + F 在某文件搜索
vscode 可以选中光标，F12 一起改 ？？待验证，群里说的。

重启 IDE CTRL + SHITF + P 键入 Reload Window

插件

1、在写 react demo 时，使用 vscode 自带的 shift + alt + f ，直接把我的 react 代码弄得很乱，后来发现格式化代码是使用插件 beautify 来弄的！后来才知道，针对 vscode 你可以安装 eslint prettier 这是为了你单独开发普通项目，你可以美化代码、校验语法，在 vue react 脚手架有的 package.json 自带给你配置了 eslint ，所以可在教手架中关闭！

2、.editorconfig 文件，此文件就是对编码方式的声明，前提是要在 IDE 中安装此插件！以前还以为 .editorconfig 是不是给 webstorm 读取的啊？后来知它是给 IDE 一个声明，让编辑器知道我的代码（无论是何种语言）风格都是按照 .editorconfig 配置。根目录建立.editorconfig 配置。这个的作用是为项目统一编辑器的实时格式。简单来说就是它负责敲代码那一刻的格式，prettier 负责保存代码那一刻的格式。两者统一体验更好。20201110 我在新奥开发分销小程序时候，发现我加了 .editorconfig 格式化就不生效了！

多项目

1、vscode 可以添加多项目，鼠标选中某目录，直接 Shift + Alt + F 可直接检索出该目录下的检索词。

杂项

1、安装了该软件，或者是任意一个工程软件，一般会在你的 Users/user 目录 （不在此目录，安装在其他目录也不合适呀） 下生成一个配置目录 .vscode ，查看之，发现安装的 vscode 插件 list 都在这！

2、用户可以根据自己需要配置 settings.json ，此文件也是类似于工程软件，一般会在你的机器用户名下的 AppData\Roaming 目录下，我个人的完整目录在 C:\Users\Andy_Zhang\AppData\Roaming\Code\User\settings.json，当然了针对你的各个不同的项目，也可单独配置，我理解：vscode 首先是读取你的项目的 .vscode\settings.json 文件，进而在读取你的系统的配置文件，属性相同的配置，合并之。若无此文件，就直接就取你系统的文件！

3、settings.json 配置了粘贴自动保存，弄巧成拙！

4、离开秋果后使用 Hasee ，vscode 在 testme 项目中写 demo ，发现格式化效果不好（注释的代码不会对齐）细看自己没有安装 prettier 插件，于是安装之，再次格式化 html 代码，提示选择格式化的程序（有一个默认的格式化程序、还一个是就是刚刚安装的 prettier ，选择之，格式化 OK！）

5、在 VSCode 中使用 Remote Workspace 进行远程文件管理 https://www.jianshu.com/p/3a18568ffa92
关于 VSCode 中工作区的讲解与使用工作区还你一个轻量的 VSCode https://www.jianshu.com/p/25706af1f6a7

6、20200820 CRS 针对 js 文件一直提示要是有分号，Google 之后发现，发现在 .eslintrc.js 中这样写 semi: [2], => semi: [0], 即可，参考：https://juejin.im/post/6844903854379630599
