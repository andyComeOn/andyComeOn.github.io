开发过程中一些警告

在最近(25.02.25)创建的 CRA 项目 react-create-app，在项目中 src\components\Hello.js 文件使用 class 写法，遇到一些警告，现记录如下：

  • 可不写构造函数及 props 和 super 调用

  • componentWillMount 提示 deprecated，UNSAFE_componentWillMount 亦提示 deprecated。建议换成 componentDidMount。

  • state 可放在 constructor 中，这是 React 官方文档推荐的方式之一。反之，亦可在类体内部直接定义 state，这种方式更为简洁。注：“如果在 constructor 中定义了 state，那么在类体内部直接定义的 state 会被覆盖”。如下：

  • 通常我们写的类方法与声明周期方法结构是平级的。如下：

    如上代码，针对按钮 onClick 事件，对应的调用函数 handleClick、triggerClick 是两种不同的形式，前者是定义的普通函数的简写形式，后者是箭头函数。二者在写法上有区别，前者需要 bind 绑定传参形式已体现，后者无需 bind 绑定，传参亦已体现。注意后者在传参时，箭头函数 () 里，当且仅当仅能写 ev 之类的形参，不能写基本数据类型的实参！

  • Do not mutate state directly. Use setState() 就是告诉 coder 记得要使用 setState 进行数据更新，不能直接操作 state！
